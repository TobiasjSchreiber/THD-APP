# THD App – UI/UX Redesign (Studienprojekt)

Dieses Projekt entstand im Rahmen des Moduls **Branded Interaction**. Ziel war die Entwicklung eines interaktiven High-Fidelity-Prototyps basierend auf dem **Human-Centred-Design (HCD) Prozess**.

## 🌐 Live-Demo & Prototyp
Die funktionale Web-App ist hier verfügbar:
👉 **[THD App Live-Link](https://tobiasjschreiber.github.io/THD-APP/)**

## 🎯 Projektziel & Fokus
Der Schwerpunkt liegt auf einer hochwertigen **User Experience (UX)** und einem begeisternden **Visual Design**. 
* **HCD-Ansatz:** Die App wurde schrittweise entwickelt, um die Bedürfnisse der Studierenden in den Mittelpunkt zu stellen.
* **Design-Fokus:** Modernisierung der THD-App durch ein intuitives und flüssiges Interface.

## ✨ Kern-Features (UX/UI)
* **Personalisiertes Dashboard:** Widgets (Mensa, Stundenplan, VPN etc.) einfach per Drag & Drop anordnen.
* **Native App-Experience & PWA:** Installierbar als Progressive Web App mit Offline-Support, Wischgesten, Fullscreen-Modus und flüssigen Animationen.
* **Responsive Design:** Optimiert für Smartphones und mit erweitertem Layout für Tablets.
* **Modernes Theming & i18n:** Unterstützung von Light-, Dark- und OLED-Mode sowie Mehrsprachigkeit (DE/EN/FI).
* **Echtzeit-Daten & Interaktivität:** Integration von iCal-Stundenplänen, der OpenMensa API sowie interaktive Diagramme (Parkplatz-Prognose, VPN-Speed, Notenverteilung).
* **KI-Integration:** Intelligente E-Mail-Antworten und Generierung realistischer Demo-Daten via Gemini-API.
* **Privacy Mode:** Auf Knopfdruck lassen sich sensible Daten wie Noten, ECTS und Guthaben zum Schutz unscharf stellen (Blur-Effekt).

## 🛠 Technische Umsetzung
* **Methodik:** Agiles Prototyping in Design-Sprints.
* **Technik:** Umgesetzt mit Vanilla HTML, CSS und JavaScript sowie einem Service Worker für PWA-Funktionen.
* **Design-System:** Konsequente Nutzung eines UI-Styleguides (feste Farben, Schriften und Komponenten).
